How does one protect the client from malicious JS attacks while still allowing arbitrary content?

Sounds like an impossible challenge, right? Well, it's not.

The trick is to create an iframe and load the content via data:-url! This way, the content has another origin than your application... and it can't access the parent via window.parent!

One other problem is that iframes can set their width to 100% - but that doesn't work for the height. So, we use the message-transfer functions... a sender in the "injected" code, and a reciever on the parent page which can update the height of the iframe element.
